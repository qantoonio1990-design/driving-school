# Изменение расписания автошколы

Все настройки расписания заданы константами в начале файла `App.jsx`.
Менять нужно **только** эти константы — остальной код использует их автоматически.

## Где что лежит (`App.jsx`, строки ~7–9)

```js
const SLOTS = ["09:00","10:30","12:00","13:30","15:00","16:30","18:00"]
const SUN_LAST = "15:00"
const OFF = [1, 3]
```

### 1. `SLOTS` — время занятий
Массив строк `"ЧЧ:ММ"` по возрастанию. Каждое занятие длится 90 минут
(см. функцию `endT`). 

- **Добавить слот:** вставь время в массив и **сохрани порядок по возрастанию**.
- **Убрать слот:** удали строку из массива.
- Формат строго `"ЧЧ:ММ"` с ведущим нулём (например `"09:00"`, не `"9:00"`).

### 2. `OFF` — выходные дни
Массив номеров дней недели: `0`=Вс, `1`=Пн, `2`=Вт, `3`=Ср, `4`=Чт, `5`=Пт, `6`=Сб.
Текущее значение `[1, 3]` = выходные по понедельникам и средам.

- **Сделать день выходным:** добавь его номер в массив.
- **Сделать рабочим:** убери номер из массива.

### 3. `SUN_LAST` — последний слот в воскресенье
В воскресенье показываются только слоты со временем `<= SUN_LAST`.
Меняй значение, если нужно укоротить/удлинить воскресный день.
Значение должно совпадать с одним из времён в `SLOTS`.

## Порядок работы

1. Прочитай текущие значения констант в `App.jsx`.
2. Внеси точечную правку только нужной константы (используй Edit, не переписывай файл).
3. Проверь сборку: `npm run build`. Она должна пройти без ошибок.
4. Кратко сообщи пользователю, что именно изменилось (старое → новое значение).
5. Закоммить с понятным сообщением и запушь в рабочую ветку.

## Важно

- Не трогай логику компонентов и БД-хелперы — только константы.
- Соблюдай сжатый стиль кода проекта (см. CLAUDE.md).
- Если правка затрагивает уже существующие записи (например, удаляется слот, на
  который кто-то записан) — предупреди пользователя, но саму базу не меняй:
  существующие записи в Supabase останутся, просто слот перестанет
  предлагаться для новых записей.
