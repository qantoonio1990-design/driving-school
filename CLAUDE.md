# Автошкола — запись на вождение

Веб-приложение для записи на занятия по вождению. Один инструктор управляет
расписанием и учениками; ученики записываются на свободные слоты по личному коду.

## Стек

- **React 18** (без TypeScript, JSX)
- **Vite 5** — сборка и dev-сервер
- **Supabase** — база данных и реал-тайм (через `@supabase/supabase-js`)
- Чистый CSS (`style.css`), без UI-библиотек
- Интерфейс на русском языке

## Структура файлов

Все файлы лежат в **корне репозитория** (без папки `src/`):

| Файл | Назначение |
|------|-----------|
| `index.html` | Точка входа HTML |
| `main.jsx` | Монтирование React-приложения |
| `App.jsx` | **Вся логика приложения** (один файл): экраны, БД-хелперы, компоненты |
| `supabase.js` | Клиент Supabase |
| `style.css` | Все стили |
| `vite.config.js` | Конфиг Vite |

## Архитектура `App.jsx`

Один файл содержит всё. Основные части (сверху вниз):

1. **Константы** — `PIN` (вход инструктора), `SLOTS` (время занятий),
   `OFF` (выходные дни: Пн и Ср), расписание воскресенья (до 15:00).
2. **DB-хелперы** (`dbLoad*`, `dbAdd*`, `dbUpdate*`, `dbDelete*`,
   `dbAddBooking` и т.д.) — все обращения к Supabase.
3. **Компоненты:**
   - `Week` — недельное расписание со слотами
   - `Login` — выбор роли + ввод PIN / кода ученика
   - `Admin` — панель инструктора (расписание + ученики)
   - `StudentView` — личный кабинет ученика
   - `App` — корневой компонент, хранит всё состояние и подписки

## Данные (Supabase)

Три таблицы:

- **`students`** — `id, name, phone, code, total_lessons, completed_lessons, created_at`
- **`bookings`** — `id, date, time_slot, student_id`
- **`blocked_slots`** — `date, time_slot` (закрытые инструктором слоты)

Синхронизация в реальном времени: подписки `postgres_changes` + резервный
опрос каждые 30 секунд.

## Соглашения и нюансы

- **Короткие имена переменных.** В `App.jsx` принят очень сжатый стиль
  (`bk`, `bl`, `sts`, `sM`, `sV` и т.п.). При правках держитесь того же стиля,
  чтобы код оставался единообразным.
- **Ключ слота** везде формируется как `` `${date}_${time}` `` — соблюдайте формат.
- **Дни недели:** 0 = Вс ... 6 = Сб. Выходные заданы в `OFF`.
- Дата хранится строкой `YYYY-MM-DD` (см. `dKey`).
- Anon-ключ Supabase в `supabase.js` — публичный (publishable), это нормально
  для клиента. Секреты в код не добавлять; реальные ключи — через `.env`
  (`VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY`).
- PIN инструктора захардкожен в коде — учитывать при правках, связанных с входом.

## Что важно при изменениях

- Сохранять русскоязычный интерфейс.
- Не ломать формат ключей слотов и формат дат.
- После правок проверять сборку: `npm run build`.
